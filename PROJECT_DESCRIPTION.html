✨ MoodMap

Mood-Based Place & Wellness Recommendation System

🎯 One-Line Summary

MoodMap is a web application that helps users track their daily emotions and provides personalized recommendations for places to visit and wellness activities based on their current mood.

📋 Project Overview

MoodMap is an innovative emotional wellness platform designed to bridge the gap between how people feel and what they can do to feel better. In today's fast-paced world, mental health awareness is crucial, and MoodMap provides a simple yet powerful tool for individuals to:

  • Track and log their daily moods with intensity levels
  • Discover nearby places (cafes, parks, gyms, etc.) that match their emotional state
  • Access personalized wellness tips and activities
  • Visualize their emotional patterns over time through analytics
  • Build healthy habits through streak-based motivation

❓ Problem Statement

Many people struggle with:

  1. Emotional Awareness: Not understanding or acknowledging their feelings
  2. Decision Paralysis: Not knowing what to do when feeling down, anxious, or stressed
  3. Lack of Resources: No easy way to find mood-appropriate activities or places
  4. Mental Health Tracking: No simple tool to monitor emotional patterns

MoodMap solves these problems by providing an intuitive platform that makes emotional wellness accessible and actionable.

💡 Key Features

Feature Description
🎭 Mood Tracking Log 5 different moods (Happy, Sad, Anxious, Calm, Energetic) with 1-5 intensity scale and optional notes
📍 Smart Places Get recommendations for nearby places (cafes, parks, gyms, temples, etc.) based on your current mood
💡 Wellness Tips Personalized activities and tips tailored to each mood type

🔄 How It Works

  1. User Registration: Create account with email and password
  2. Log Mood: Select one of 5 moods and set intensity (1-5)
  3. Get Recommendations:
    • Mood-matched places appear automatically
    • Wellness tips update based on current mood
  4. Explore Places: Browse places by city and area, get directions
  5. Track Progress: View analytics, maintain streaks, see patterns

❓ Frequently Asked Questions (For Judges/Visitors)

Q1: What is MoodMap?

MoodMap is a mood tracking and wellness recommendation web application. Users log how they're feeling, and the app suggests places to visit and activities to do based on their current emotional state.

Q2: Why is this project useful?

Mental health awareness is crucial today. Many people don't know what to do when they feel anxious, sad, or stressed. MoodMap bridges this gap by providing actionable recommendations - whether it's visiting a calm park when anxious or an energetic gym when feeling low.

Q3: What technologies did you use?

Frontend: HTML, CSS, JavaScript with Tailwind CSS for styling. Backend: PHP with MySQL database. Server: Apache via XAMPP. We also used Chart.js for analytics visualization.

Q4: How does the mood-place matching work?

Each mood type has associated place categories. For example, "Anxious" mood suggests calming places like parks, temples, and libraries. "Happy" mood suggests social places like cafes, restaurants, and malls. The algorithm matches mood → place type → available places in the user's selected area.

Q5: Is this using real location data?

Currently, we have pre-loaded place data for 10 major Indian cities with multiple areas each. In production, this can be integrated with Google Places API for real-time data.

Q6: How is user data secured?

Passwords are hashed using PHP's password_hash() with bcrypt algorithm. All database queries use prepared statements to prevent SQL injection. Sessions are secured with httpOnly cookies.

Q7: What makes this project unique?

Unlike simple mood trackers, MoodMap is action-oriented. It doesn't just track moods - it helps users DO something about how they feel. The combination of mood tracking + place recommendations + wellness tips in one platform is unique.

Q8: Can this be deployed commercially?

Yes! With Google Places API integration for real-time places, cloud hosting, and a mobile app version, MoodMap can become a full commercial product. Future monetization could include premium features, therapist partnerships, and wellness brand collaborations.

Q9: What challenges did you face?

Main challenges included: (1) Designing an intuitive mood selection UI, (2) Creating meaningful mood-to-place mappings, (3) Building responsive animations that work smoothly, and (4) Structuring the database for efficient mood analytics queries.

Q10: What did you learn from this project?

Full-stack web development with PHP and MySQL, modern CSS techniques like glassmorphism and animations, database design for analytics, user authentication best practices, and the importance of UX design in wellness applications.
